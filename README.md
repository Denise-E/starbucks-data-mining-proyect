# Starbucks Data Mining Project

Data Mining analysis of Starbucks customer ordering patterns using Decision Trees and Clustering (WEKA + Python).

**Universidad de Palermo** — Análisis de la Información y la Decisión

---

## Business Problem

Building on our previous Data Warehouse project, we identified that **Drive-Thru is the main operational bottleneck** (5.79 min avg vs. 3.22 min baseline). This project goes one step further: instead of describing what happened, we use Data Mining to **predict and segment**:

- **Decision Tree:** What factors determine whether a customer ends up satisfied or dissatisfied?
- **Clustering:** What behavioral customer profiles exist, and how do they differ?

```
starbucks-data-mining-proyect/
│
├── data/

│
├── notebooks/
```

## Methods

| Method | Algorithm | Tool | Target |
|---|---|---|---|
| Decision Tree | J48 (C4.5) | WEKA | `customer_satisfaction` → High (≥4) / Low (≤3) |
| Clustering | SimpleKMeans | WEKA | Customer behavioral segments |

**Key features used:** `order_channel`, `fulfillment_time_min`, `cart_size`, `num_customizations`, `has_food_item`, `is_rewards_member`, `store_location_type`, `time_period`

## How to Run

**1. Prerequisites**

Make sure you have **Python 3.11 or higher** installed. You can check your version with:

```bash
python --version
```

If not installed, download it from [python.org](https://www.python.org/downloads/).

**2. Create and activate the virtual environment**

```bash
python -m venv venv
venv\Scripts\activate        # Windows
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Run the pipeline**

To run the full EDA + preprocessing flow and generate the ARFF files:

```bash
python main.py
```

Or explore each stage interactively:

```bash
jupyter notebook
```

Open and run in order:
1. `notebooks/01_eda.ipynb`
2. `notebooks/02_preprocessing.ipynb` — generates `weka/decision_tree.arff` and `weka/clustering.arff`
3. `notebooks/03_results_analysis.ipynb` — *(pending: run after WEKA models)*

**5. Run the WEKA models**

- Open WEKA Explorer
- Load `weka/decision_tree.arff` -> Classify tab -> J48 -> Start
- Load `weka/clustering.arff` -> Cluster tab -> SimpleKMeans -> Start
- Save outputs to `weka/results/`
